## 1. 프로젝트 개요 (Project Overview)
- **목표**: 횡스크롤 게임 요소가 포함된 인터랙티브 모바일 청첩장 개발.
- **핵심 컨셉**: 2018년 첫 만남(독산역)부터 현재까지의 여정을 캐릭터의 이동과 시간 변화로 시각화.
- **주요 기술**: Android (Native Wrapper) + Phaser 3 (Web Game Engine).

## 2. 시스템 아키텍처 (Architecture)

### 2.1 Android Wrapper (`/android`)
- **언어**: Kotlin
- **주요 컴포넌트**: `MainActivity.kt`
- **구현 특징**:
    - `WebView`를 통한 로컬 게임 콘텐츠(`assets/www/index.html`) 로드.
    - **Edge-to-Edge**: 상태 표시줄 및 내비게이션 바를 숨긴 전체 화면 모드.
    - **Chrome Remote Debugging**: PC 크롬 브라우저에서 `chrome://inspect`를 통해 디버깅 가능.
    - **Console Log Bridge**: JavaScript의 `console.log`를 Android Logcat(`WebViewConsole` 태그)으로 전달.
    - **CORS 설정**: 로컬 파일 간 접근 허용 (`allowFileAccessFromFileURLs` 등).

### 2.2 Web Game Engine (`/android/app/src/main/assets/www`)
- **엔진**: Phaser 3 (CDN 로드)
- **구조 (OOP 기반)**:
    - `index.html`: 게임 컨테이너 및 UI 레이아웃.
    - `main.js`: Phaser 씬(Scene) 관리 및 게임 루프.
    - `js/config.js`: 모든 상수, 에셋 경로, 물리 엔진 설정, 대화 데이터 집약.
    - `js/entities.js`: `Groom`, `Bride` 캐릭터 클래스 (물리 및 애니메이션).
    - `js/ui.js`: 가상 조이스틱 및 버튼(JUMP, TALK) 구현.
    - `js/dialogue.js`: 대화창 및 선택지 시스템.
    - `style.css`: 게임 오버레이 UI 스타일링.

## 3. 주요 구현 기능 (Key Features Implemented)

### 3.1 캐릭터 시스템
- **Groom (신랑)**: 조이스틱/키보드로 이동 및 점프 가능. 카메라가 항상 추적.
- **Bride (신부)**: 특정 지점에 대기 중이며, 일정 거리 내 접근 시 느낌표(!) 표시. 상호작용 성공 시 신랑을 따라다님.
- **Real-world Scaling**: 캐릭터의 실제 키(cm)를 기준으로 화면 내 크기를 자동 계산하여 렌더링.

### 3.2 시간 게이지 시스템
- 신랑의 X 좌표 이동 거리에 비례하여 `2018-02-15`부터 날짜가 실시간으로 흐름.
- `GAME_CONFIG.PHYSICS.PIXELS_PER_DAY` 설정을 통해 날짜 흐름 속도 조절 가능.

### 3.3 조작 체계
- **Mobile First**: 가상 조이스틱(좌측)과 JUMP/TALK 버튼(우측) 제공.
- **Keyboard Support**: 방향키로 이동 가능.
